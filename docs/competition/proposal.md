# MoonModGuard 项目申报书

## 项目名称

MoonModGuard：MoonBit 项目清单与供应链策略审计器

## 项目简介

MoonModGuard 是一个面向 MoonBit 生态的项目清单与供应链策略审计工具。项目解析 `moon.mod`、`moon.pkg` 等项目清单文本，提取模块元数据、包级依赖、导入别名和发布信息，构建项目快照，并根据可配置策略输出元数据完整性、许可证合规性、依赖重复和未知依赖等诊断结果。

## 项目方向与适用场景

MoonBit 生态正在积累越来越多的包和工程模板。包能否被复现、发布和维护，不只取决于源码是否能编译，也取决于清单元数据是否完整、依赖声明是否清晰、许可证是否符合使用策略。MoonModGuard 将这些工程质量要求抽象成可测试的审计规则，为 MoonBit 包发布、教学检查、竞赛仓库验收和后续 CI 集成提供基础组件。

适用场景包括：

- MoonBit 包发布前的清单质量检查；
- GitLink 竞赛仓库的元数据与依赖审计；
- 课程或团队项目的统一工程规范检查；
- 后续扩展为包仓库、CI 或软件供应链分析工具。

## 计划实现的核心功能

- 解析 `moon.mod` 中的项目名称、版本、许可证、README、仓库地址、关键词和描述。
- 解析 `moon.mod` 与 `moon.pkg` 中的 import 依赖和导入别名。
- 构建项目模型，合并模块级和包级依赖。
- 根据策略检查缺失元数据、非 allowlist 许可证、重复依赖和未知依赖前缀。
- 输出结构化审计结果和 Markdown 报告。
- 提供命令行演示、README、测试、设计文档和 CI 配置。

## 原创性说明

本项目为原创项目，不移植已有开源项目。项目选题聚焦 MoonBit 自身工程元数据和包供应链策略审计，不与通用数据结构、内容寻址、解释执行、差异算法或常见格式解析方向重合。首版不引入外部依赖，不做平台服务绑定，优先形成可测试、可复用、可扩展的审计内核。

## 技术路线

项目采用轻量扫描器解析清单文本中的字符串字段、数组字段和 import 块，生成 `ModuleManifest`、`PackageManifest` 与 `ProjectModel`。策略评估器以 `Policy` 为输入，检查必填元数据、许可证 allowlist、依赖前缀和重复依赖，并生成 `AuditReport`。报告渲染器将审计结果转换为稳定的 Markdown 文本，便于 CLI、CI 和文档系统复用。

## 预期成果

项目预期交付一个独立的 MoonBit 审计基础库，覆盖清单解析、依赖提取、策略检查和报告输出的测试集，提供可运行 CLI 示例、README、竞赛申报材料和可在 GitLink 复现的完整仓库。后续可扩展真实文件扫描、工作区遍历、依赖图可视化和包仓库审计。
