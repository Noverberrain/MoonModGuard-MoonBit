# MoonModGuard 开发报告

## 开发过程

MoonModGuard 是一个 MoonBit 项目清单与供应链策略审计工具。通过解析 `moon.mod` 和 `moon.pkg` 文件，验证项目元数据的完整性和合规性，检查依赖项是否符合预定义的安全策略。

开发分为三个阶段：
1. **解析阶段**：实现 moon.mod 和 moon.pkg 文件的解析器，提取项目名称、版本、许可证、依赖等信息
2. **审计阶段**：实现策略检查引擎，验证许可证合规、仓库地址存在性、README 完整性等
3. **报告阶段**：生成审计报告，标记不合规项及其位置

## 架构设计

```
moon.mod / moon.pkg 文件
  → Parser（解析器）
  → ModuleManifest + PackageManifest
  → ProjectSnapshot（项目快照）
  → Policy（策略定义）
  → Auditor（审计引擎）
  → 审计报告（合规/不合规 + 位置信息）
```

模块划分：
- 主模块：解析、数据模型、审计逻辑均在 `moonmodguard.mbt` 中
- 工具链：`cmd/main` CLI 入口
- 脚本：`scripts/` 辅助工具

## 技术难点

### 1. moon.mod 格式解析
moon.mod 使用 TOML-like 格式但并非标准 TOML。解析器需要处理 MoonBit 特有的 `import { ... }` 语法块和注释格式，通过逐行解析和状态机模式提取结构化信息。

### 2. 策略引擎设计
策略检查需要支持多种规则组合（许可证白名单、可信前缀、README/仓库必填等）。通过 Policy 结构体统一管理所有检查项，审计引擎逐项验证并汇总结果。

### 3. 位置追踪
对于 moon.mod 文件中的不合规项，需要精确定位到具体行号和列号。Position 结构体记录 line/column，错误报告时附带源码位置方便修复。

## 不足与展望

1. 支持的策略类型有限（许可证、仓库、README），可扩展更多检查项
2. 不支持 moon.pkg 的深度依赖分析
3. 报告格式仅支持文本输出，可扩展 JSON/HTML
4. 未实现自动修复功能
